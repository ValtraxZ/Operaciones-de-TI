Introducción

En el contexto actual del desarrollo de software, donde la escalabilidad, la resiliencia y el rendimiento son aspectos críticos para el éxito de una aplicación, contar con un sistema de monitoreo robusto se ha vuelto indispensable. Este proyecto se centra en la implementación de un entorno de monitoreo integral para una aplicación construida con Spring Boot, un framework ampliamente utilizado en el ecosistema Java para el desarrollo de microservicios y aplicaciones empresariales, aprovechándose de las capacidades del módulo Spring Boot Actuator, el cual expone endpoints HTTP con información detallada sobre la salud, métricas y configuración de la aplicación. Esta integración facilitó la conexión entre la aplicación y las herramientas de monitoreo, garantizando un flujo de datos eficiente y preciso.

Objetivos

Implementar un sistema de monitoreo en tiempo real para una aplicación Spring Boot utilizando Prometheus como herramienta de recolección y almacenamiento de métricas, y Grafana para la visualización de datos.
Desarrollar dashboards interactivos en Grafana que permitan visualizar de manera clara y consolidada el estado de la aplicación, facilitando la identificación de patrones, tendencias y comportamientos anómalos.
Configurar alertas automáticas mediante Prometheus o Grafana para notificar de manera proactiva sobre eventos críticos, como caídas en el rendimiento, errores recurrentes o consumo excesivo de recursos.

Diseño de la Solución

Aplicación Spring Boot
– Expone endpoints HTTP con métricas de la aplicación.
– Se configura para exponer métricas en formato Prometheus.

Prometheus
– Recolecta métricas desde el endpoint /actuator/prometheus de la aplicación Spring Boot.
– Configuración mediante un archivo prometheus.yml donde se definen el intervalo de recolección de datos y el puerto de acceso.

Grafana
– Conecta con Prometheus como fuente de datos.
– Crea dashboards personalizados con paneles para las métricas de JVM y uso de recursos del sistema.
– Configura un sistema de alertas para enviar notificaciones por correo mediante Gmail.

Implementación

1. Se agregan las dependencias en Spring Boot para recolectar las métricas (Actuator y Micrometer).
2. Se configuran los endpoints del Actuator en el archivo application.properties.
3. Se crea un archivo prometheus.yml que define el job de monitoreo, la ruta del endpoint y el puerto.
4. Se elabora el archivo docker-compose.yml para descargar las imágenes de Grafana y Prometheus, configurando puertos, credenciales de acceso en Grafana y parámetros para alertas (correo remitente y receptor).
5. Con el comando docker compose up se inicia el entorno: Prometheus se verifica en el puerto 9090 para asegurar que las métricas se recolectan correctamente.
6. Se accede a Grafana en el puerto 3000 para establecer la conexión con Prometheus y extraer la información.
7. Finalmente, se crea un dashboard en Grafana para la visualización continua de las métricas.

Pruebas y Resultados

Durante una ejecución con baja actividad, se observó que la aplicación presenta valores bajos en el uso de memoria, lo que indica un consumo moderado. Adicionalmente, se comprobó que el Garbage Collector de la JVM opera de forma cíclica y eficiente, gestionando adecuadamente los objetos.
Para evaluar el rendimiento bajo estrés, se desarrolló un método que crea hilos de manera continua. Esto activó la alerta configurada, notificando por correo un elevado uso de CPU. En el dashboard se verificaron los niveles altos de CPU, lo que confirma la respuesta del sistema ante el incremento de la carga y el adecuado manejo de la JVM por parte del Garbage Collector.

Implementación

Configuración de Spring Boot (Actuator y Micrometer), Prometheus (prometheus.yml) y docker-compose para integrar Grafana y Prometheus sin sobrecargar la vista.

Conexión y Dashboards

Se establece la conexión entre Grafana y Prometheus y se crean dashboards interactivos para visualizar métricas del sistema y de la JVM.

Pruebas y Alertas

Pruebas en entorno de baja actividad y estrés. Se activan alertas de uso elevado de CPU y se verifica el rendimiento de la garbage collection.

Conclusiones

La implementación del sistema de monitoreo utilizando Spring Boot, Prometheus y Grafana ha permitido obtener una visión detallada del rendimiento y uso de recursos de la aplicación. Las pruebas realizadas demuestran que, aun bajo estrés, el mecanismo de garbage collection opera eficazmente, evitando saturaciones y permitiendo que las alertas se activen de manera oportuna para la toma de decisiones proactivas.
